# RIVERWOOD PROJECTS LLP
## AI Systems Internship Challenge
### AI Voice Agent Prototype

**Why This Challenge**
Our AI Voice internship announcement received more than 1000 applications within a few hours. Instead of traditional interviews, Riverwood prefers builders who can demonstrate execution. This challenge helps us identify candidates who can design and build real AI systems.

**About Riverwood**
Riverwood Projects LLP is a next generation real estate development company focused on plotted townships under the Deen Dayal Jan Awas Yojana (DDJAY). Our flagship project, Riverwood Estate in Sector 7 Kharkhauda, is a 25 acre residential township located near the upcoming IMT Kharkhauda industrial hub anchored by Maruti Suzuki. Our philosophy is simple: Building Foundations and creating long term relationships with our customers.

**The Vision**
Riverwood is building an AI powered CRM system for real estate. One key component is a human like AI voice agent that calls customers with personalized updates about construction progress and project milestones while maintaining a warm conversational tone.

**Your Challenge**
* Build a simple prototype of an AI voice agent for Riverwood.
* The agent should greet the user naturally in Hindi or English.
* It should accept user input by voice or text.
* It should generate contextual responses using an LLM.
* It should convert responses into natural speech.
* Bonus: The agent remembers previous conversation context.

**First Level Submission**
Candidates must submit a Loom screen recording showing the working prototype. The recording should demonstrate the interaction flow between the user and the AI agent and briefly explain the system architecture.

**Example Scenario**
* AI calls the user.
* Greets the customer.
* Shares a construction progress update.
* Asks if the customer plans to visit the site.
* Records the response.

**API Support for the Challenge**
If required, Riverwood can provide paid API keys including OpenAI, ElevenLabs, Twilio, VAPI, and Apollo. Candidates who require access to these services may send a request email.

**Recommended Tech Stack**
* OpenAI GPT models
* ElevenLabs or PlayHT for voice synthesis
* Twilio Voice or VAPI
* LangChain or Vocode
* Python or Node.js
* Replit or GitHub

**Technical Thinking Question**
In your submission note answer this question: If Riverwood needs to call 1000 customers every morning, how would you design the infrastructure to handle these calls efficiently?

**Submission Requirements**
* Loom video showing the working prototype
* Demo link such as GitHub, Replit, or Drive
* Short technical note explaining architecture
* Estimated infrastructure cost per 1000 calls
